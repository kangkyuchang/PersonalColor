# 프로젝트 개요
### 해결하려는 문제 정의
기존의 퍼스널 컬러 진단 방식은 전문 컨설턴트의 **주관적인 판단**과 **시간/비용**을 필요로 합니다.
또한, 진단을 위해서는 **살제 드레이프(천)를 얼굴에 직접 대보는 과정이**이 필수적이므로,
사용자는 시간과 장소의 제약을 받으며 진단 결과의 **객관성**과 **일관성**을 확보하기 어렵습니다.

### AI/딥러닝 선정 이유
퍼스널 컬러 진단은 얼굴의 **미묘한 톤, 명도, 채도, 대비** 등 복잡한 시각적 특징을 **정밀하게 분석**하고 **색상 팔레트와 매칭**해야 하는 고차원적인 작업입니다.

일반적인 프로그래밍 로직으로는 이러한 **미묘한 시각 패턴과 피부 톤의 변화를 정량적으로 인식하고 분류**하는 것이 불가능합니다. 
따라서 **딥러닝 기반의 이미지 인식 및 분석 기술**을 활용하여:
1. 사용자의 **사진 한 장**만으로 얼굴 톤 영역을 정확하게 식별하고,
2. 전문가의 진단 결과를 **객관적인 데이터**로 학습하여,
   **정확도가 높은 자동화된 퍼스널 컬러 진단 시스템**을 구축하고자 했습니다.

### 최종 목표 및 평가 지표
본 프로젝트는 전문적인 컨설팅 없이도 신뢰할 수 있는 결과를 제공하는 **퍼스널 컬러 진단 AI 모델 개발**을 목표로 합니다.
| 목표 항목 | 핵심 내용 |
|:---|:----|
| **문제 해결 목표** | **사용자의 사진 한 장**만으로 퍼스너 컬러(예: 봄 웜톤, 겨울 쿨돈 등)를 진단 |
| **성능 목표** | 전문가의 진단결고와 비교하여 **높은 정확도** 달성 |
| **핵심가치** | **언제 어디서든** 제약 없이 자신의 퍼스널 컬러를 **객관적이고 일관된 결과**로 확인 가능한 서비스 제공 |

| 구분 | 내용 | 상세 설명 |

| **이미지 유형** | **연예인 얼굴 이미지** | 다양한 조명과 환경에서 촬영된 인물의 얼굴 이미지를 활용하여 모델의 **일반화 능력**을 확보 |

### 2. 데이터 전처리 및 레이블링
- **레이블링 방식**: CSV 파일을 통해 각 이미지에 대해 **'fall', 'spring', 'summer', 'winter'** 중 **단 하나의 계절 컬러**에 해당하는 열에 1 표시되는 **다중 클래스 분류** 형태로 레이블을 부여
- **전처리**: 모델 입력에 적합도록 이미지 크기 조정 및 정규화 처리 진행

# 모델링 및 실험 (Modeling & Experimentation)
### 1. 모델 아키텍처 선정 및 설계
- **모델 선정: CNN(Convolutional Neural Network)**
    - **선정 이유**: 얼굴 이미지의 톤, 명도, 대비 등 **공간적 특징**을 효과적으로 추출하고 계층적으로 학습하는 데 가장 적합한 모델입니다.
- **모델 구조**:
    - **입력 계충(Input Layer)**: `(416, 416, 3)` (가로 416px, 세로 416px, 3채널 RGB 이미지)
    - **특징 추출 계층(Convolution & Pooling)**: `Conv2D`와 `MaxPooling2D`를 3단계로 반복하여 이미지의 **지역적 특징**을 점진적으로 추출하고 차원을 축소했습니다.
        - 필터 수: 32→64→128로 증가시켜 심층적인 특징을 학습하도록 설계
    - **완전 연결 계층(Dense Layer)**: 추출된 특징을 기반으로 최종 분류를 수행합니다.
    - **출력 계층(Output Layer)**: `Dense(4)`와 `softmax` 활성화 함수를 사용하여 4개의 퍼스널 컬러 클래 중 하나의 확률를 출력합니다.
 
### 2. 학습 전략 및 튜닝
| 요소 | 설정 값 | 목적 및 해석 |
|:---|:---|:---|
| **옵티마이저(Optimizer)** | `adam` | 경사 하강법 기반의 효율적이고 보편적인 최적화 기법을 사용하여 모델 학습 속도 및 성능 안정화 |
| **손실 함수(Loss Function)** | `sparse_categorical_crossentropy` | 4개의 클래스 중 하나로 분류하는 **다중 클래스 분류 문제**에 적합하며, 레이블이 정수 인코딩되어 있을 때 효율적 |
| **평가 지표** | `accuracy` | 전체 데이터 중 모델이 정확하게 분류한 비율을 측정하여 최종 목표인 '높은 정확도'를 직접적으로 확인 |
| **에포크(Epochs)** | 10 | 학습 횟수를 10으로 설정하여 초기 성능을 확인하고, 과적합 여부를 신속히 파악 |
| **배치 크기(Batch Size)** | 32 | 한 번의 가중치 업데이트에 사용되는 샘플 수를 32로 설정하여 학습의 안정성과 효율성 균형 유지 |
| **규제(Regularization)** | `Dropout(0.2)` | 완전 연결 계층에서 20%의 뉴런을 무작위로 비활성화하여, 모델이 특정 특징에 과도하게 의존하는 과적합을 방지하도록 설계 |
